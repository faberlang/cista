# Cista v0.1.0

First public **component release** of the Cista package-store CLI and library.
Source tag `v0.1.0` matches `Cargo.toml` `version = "0.1.0"`. Public downloadable
artifacts publish to `faberlang/releases` as `cista-v0.1.0` (multi-arch).

This is the first tagged binary release for standalone `faberlang/cista` (no
prior `cista-v*` artifacts on the shared release surface).

## Product surface

Cista is the **package manager** layer for Faber: install, resolve, inspect,
cache, run installed binaries, and registry login/fetch/publish. It does **not**
depend on Radix or the compiler.

| Tool | Role |
| --- | --- |
| `faber` | Project workflow (check/build/run/test); may call Cista APIs |
| `cista` | Package store CLI (`$CISTAE_HOME`, default `~/.faber/cistae`) |
| `radix` | Private compiler (not a Cista dependency) |

### CLI commands (v0.1.0)

`init`, `check`, `inspect`, `metadata`, `graph`, `resolve`, `fetch`, `install`,
`run`, `remove`, `update`, `cache`, `package`, `runtime`, `target`, `publish`,
`yank`, `login`, `logout`, `doctor`.

## Major tracks

### Local package store (Phase A and follow-ons)

- Local library store operations and `faber.lock` rewrite path.
- Install into the shared store; remove with identity checks.
- Run executables from installed binary packages (host entry materialization).
- Interfaces-only packages and Norma platform-default lock support.
- Install API exposed for the product CLI (`feat(cista): expose install API…`).

### Durability and concurrency

- Atomic staged installs; crash-durable critical sections.
- Store mutation locking (install/remove serialized).
- Meta-install expand + rollback of partial installs; commit-aware rollback.
- Stale install transaction directories hidden from discovery.
- Reserved cache namespace; reject transaction-suffix versions.

### Path safety and archive hardening

- Contain package paths; reject special entries, non-UTF-8 paths, symlink roots.
- Reject recursive/unsafe publish destinations and overlapping package copies.
- Registry: reject unsafe archive entries (inbound and remote); verify package
  identity on fetch and local publish.
- Reject dangerous archive modes.

### Registry client (hermetic contract)

- HTTPS/bearer `login` / `logout`.
- `fetch --registry-url` and `publish --registry-url`.
- Live cista.dev validation remains environment-gated (not required for this
  binary release).

### Quality / CI

- GitHub Actions release workflow (`.github/workflows/release.yml`) builds
  linux-x64, darwin-x64, darwin-arm64 and publishes to `faberlang/releases`.
- Clippy pedantic deny; hygiene ratchet; expanded unit/CLI/install tests.

## Publish

1. Push `main` (includes this notes file if committed).
2. Annotated tag on the release commit:

   ```bash
   git tag -a v0.1.0 -m "Cista v0.1.0"
   git push origin v0.1.0
   ```

3. Workflow `Release Cista` builds multi-arch archives and creates/uploads
   `cista-v0.1.0` on `faberlang/releases`.

## Out of scope for 0.1.0

- Live production proof against cista.dev (campaign under
  `docs/factory/cista-dev-registry/`).
- Homebrew / installer channels (shared release repo only for now).
- Version bump beyond `0.1.0` (crate already at first public semver).
